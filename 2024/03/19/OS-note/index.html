操作系统学习笔记

发表于2024-03-19|更新于2024-03-22
|阅读量:

第一章 绪论

知识点总结

操作系统的特性

  1. 并发性

    并发性对应并行性，先说二者区别

    并发：两个或以上的事件在同一时间间隔 发生，即宏观上多个程序在并行运行，微观上多个程序串行运行

    并行：两个或以上的事件在同一时间点上发生，实质是有多个CPU或者CPU有多核结构 的硬件支持，无论宏观还是微观都是并行的

    并发性的好处：让CPU和IO设备同时执行，提高资源的利用率

  2. 共享性

    多个并发的程序使用同一个计算机资源，分为分时间的共享（轮流使用）和分空间的共享（各自用一部分）

    并发性和共享性是现代操作系统最基本的特性

  3. 虚拟性

    指在物理上没有提供，但逻辑上具备的功能，如：

    虚拟化处理机，让多个程序并发执行，分时使用一个CPU，一个处理机同时为多个用户服务，且每个用户都认为有一个处理机处理其需求内容（时间片，并发）

    虚拟化内存，逻辑上扩充存储器内容，达到内存扩充的效果（请求调入和置换）

    虚拟化设备，让一台IO设备能够被多个用户同时访问，并且使用逻辑设备名（缓存，共享）

  4. 不确定性（异步性、随机性）

    是并发性和共享性的必然结果，让每个进程的运行过程不再“一气呵成”，但是能够保证无论事件在什么时候、以什么次序、何种方式发生，其处理结果都是相同的

操作系统的接口

操作系统主要想用户提供了命令接口和程序接口，程序接口也就是常考的系统调用，此外还有图形接口，这是调用了系统调用实现的功能。广义指令就是系统调用的命令

系统调用的面向对象是程序或者编程人员，只能通过用户程序间接使用

访管

访管指令是一条可以在用户态（又称目态）下执行的指令。在用户程序中，因要求操作系统提供服务而有意识地使用访管中断，从而产生一个中断事件（自愿中断），将操作系统转换为核心态，称为访管中断。

访管中断由访管指令产生，程序员使用访管指令向操作系统请求服务。

访管指令仅能在用户态使用，属于非特权指令，且执行访管指令会将CPU从用户态变为核心态，且核心态的CPU只能执行访管以外的全部指令

中断

中断分为外中断与内中断（异常），单出现中断时无特别说明常指外中断，常见的中断有如下几种：

中断 IO设备结束的中断（包括DMA、通道等） 时钟中断

异常 程序操作非法 地址越界 缺页（故障） 陷入指令（自陷） 硬件异常（终止）

宏内核与微内核

这是从操作系统的内核架构来划分的：

宏内核：又称单内核、大内核，主流操作系统如Windows，Linux，Android，macOS 等都是宏内核结构。该结构将所有功能模块都作为一个紧密联系的整体放在核心块，性能高，很少在用户态和核心态进行切换

微内核：随着对操作系统的需求增加，微内核优势越来越明显，其 扩展性高、系统可靠性与可移植性强、支持分布式系统、采用了面向对象的技术，但相对的需要频繁在用户态和核心态之间切换，开销大

操作系统开机步骤

激活CPU—硬件自检—加载系统盘—加载主引导记录（MBR）—扫描分区盘—加载分区引导记录（PBR）—加载启动管理器—加载操作系统

简答题

  1. 现代操作系统一般要满足哪些主要的设计目标？

    （1） 系统效率，操作系统的效率主要体现在对资源，特别是是CPU以及内存等高速设备，的利用率高，用户任务的周转时间和响应时间尽可能短，系统吞吐量（单位时间处理的任务数量）尽可能大

    （2） 系统安全性和可靠性，网络信息时代，操作系统作为计算机系统中所有硬件、软件资源的管理者必须要有足够的可靠性和安全性

    （3）系统可维护性，操作系统是否拥有较好的维护性是决定操作系统生命周期长短的重要因素

    （4）系统易用性，体现在操作界面和编程接口的简洁、方便、易用

    （5）可扩充性，体现在操作系统的功能能够被不断加强、改进和完善，引入新组建时能够不影响原有组件的功能

    （6）开放性，即能够与不同厂家制造的计算机相关设备及其各种应用程序有效协同工作

  2. 怎样理解操作系统的作用

    用户角度：用户角度看，操作系统是用户与计算机硬件系统之间的接口，包括以下两种类型

    ​ 操作接口：图形化界面或命令行窗口，能够对系统资源进行操作

    ​ 编程接口：为程序员使用的应用程序提供编程接口，能够调用模块实现程序功能

    资源管理角度：从资源管理角度看，操作系统是系统所有资源的管理者，包括硬件资源、软件资源、数据资源等。计算机一切资源都需要考操作系统进行管理

  3. 操作系统的定义与主要特性

    定义

    • 现代操作系统是计算机系统中最为基本的一种系统软件，是程序模块的集合。

    • 操作系统能够以尽量有效、合理的方式组织和管理计算机的软、硬件资源，合理地组织计算机的工作流程，控制程序的执行

    • 能够向用户提供各种服务功能，使计算机系统能够高效地运行

    • 能够改善人机界面，使用户能够灵活、方便、有效地应用计算机

    特性

    • 并发性：一个时间段内同时运行多个进程，让CPU和IO设备同时执行，提高资源的利用率

    • 共享性：让多个并发的程序使用同一个计算机资源

    • 虚拟性：在逻辑上实现物理上不满足的功能，将物理实体变成若干个逻辑上的对应物

    • 不确定性：进程以人们不可预知的速度向前推进，只要运行环境相同，结果必然相同

  4. 分别叙述多道批处理系统、分时操作系统、实时操作系统的特点

    OS的基本类型分为批处理系统、分时操作系统、实时操作系统，批处理系统又分为单道批处理系统和多道批处理系统：

    多道批处理系统——对资源的使用依次进行，我用完了你接着用

    • 多道：计算机内存同时存放多道相互独立的程序（与之对应的就是单道批的单道）

    • 宏观上并行：同时进入系统的多道程序都是运行的，先后开始各自的运行但都未运行完毕就换到了下一个程序

    • 微观上串行：多道程序实质上是轮流占用CPU，交替地执行

    分时操作系统——时间片

    • 同时性：也称多路性，允许多个用户能够同时使用一台计算机

    • 交互性：用户能够通过人机对话的方式直接控制程序运行，与程序进行交互

    • 独立性：各个用户之间独立地进行操作，不会感受到别人也在用这个计算机

    • 及时性：用户能够在很短的时间内获得计算机的响应

    实时操作系统——能够在极短的时间内完成某些紧急任务而不需排队

    硬实时操作系统：必须在某个时间范围内完成的——自动控制系统，导弹系统、、

    软实时操作系统：能够偶尔违反时间规定而不会造成过大损失——订票系统、管理系统、、

    • 及时性：能够在很短的时间范围的就能完成响应

    • 可靠性：紧急任务可不需要排队，优先完成以保证程序顺利进行（抢占式优先级）

  5. 如何理解多道程序系统中的“内存中多个程序执行过程交织在一起，各个进程都在走走停停”的现象

    这一现象是多道程序系统中为了确保设备能够尽可能最大化地利用，让设备在可被使用且被需要时立马就能用上，这样可以提高CPU的利用率，减少了CPU的等待时间，达到了并发的目的，即宏观上实现并行，微观上为串行

第二章 处理器管理

知识点总结

进程的特性

参考教材《现代操作系统教程》

  1. 并发性：进程的并发性与操作系统对应，指的是多个进程并发地在处理器中运行

  2. 共享性：同样与操作系统的共享性对应，多个进程在系统中共享各种资源，包括CPU、内存，IO设备等

  3. 结构性：指进程由多个方面构成，分别为程序、数据、进程控制块（PCB）和堆栈

  4. 动态性：进程的状态是动态变化的，总体流程为创建-运行-灭亡

  5. 独立性：并发运行的多个进程是批次独立的，在组成结构中没有和其他进程有所交集，使用系统分配的资源彼此独立地推进

  6. 制约性：各自独立的个体可能会有资源的竞争（进程互斥）或者协作（进程同步与通信），这并不影响进程的独立性，但也意味着一个进程的运行会受到另一个进程等等制约

参考教材《王道》

  1. 动态性

  2. 并发性

  3. 独立性

  4. 异步性：由于进程之间相互制约，是的进程的推进不可预知，各自独立，可能会导致运行结果的不可再现——进程同步机制的必要性

进程状态

三状态模型（最基本）

  1. 运行态：当前进程占用CPU并正在运行

  2. 就绪态：当前进程已经具备了在CPU上执行的其他条件，但还未被分配CPU

  3. 等待态：又称阻塞态、睡眠态、封锁态，当前进程正在等待某件事的发生，或是某个信号、资源以满足运行条件，此时就进入等待态

相互转换

  1. 就绪->运行：CPU空闲且通过调度算法选中该进程

  2. 运行->就绪：CPU分配的时间片用完 或 被更高优先级的进程抢占

  3. 运行->等待：运行过程中申请资源但是系统暂时无法满足 或 进程需要等待某个信号或者系统提供某项服务（如IO操作）

  4. 等待->就绪：所等待的资源可以被提供

五状态模型——在基础的三状态模型的基础上添加了创建态与结束态

  1. 创建态：进程正在被创建，但还没经历完这些过程：申请PCB—填写进程控制与管理信息—分配进程运行所需资源—初始化PCB—将PCB挂入就绪队列—返回进程标志号

  2. 结束态：也称终止态，进程正在消失，可能是正常结束，也可能是其他必须终止的原因导致进程无法继续进行（除数为0），设置为结束态后开始释放与回收资源

新增转换

  1. 无->创建态：新创建进程就能从无到有

  2. 创建->就绪：完成创建工作且当前资源能够满足运行

  3. 运行->终止：进程自然运行结束/出现无法克服的异常错误/被操作系统、用户、父进程强行终止

  4. 就绪/等待->终止：被操作系统、用户、父进程强行终止

  5. 终止态->无：完成善后工作，资源释放、回收完成

七状态模型——添加就绪挂起和等待挂起

  1. 挂起：当前系统所剩余的资源已经无法满足所有进程的需求时，将某些程序、数据迁移到外存中，释放其原本占有的资源

    当前就绪态进程为空或者就绪态中的某些进程要求更多的资源时，系统就会挂起部分进程

  2. 挂起——激活；等待——唤醒

进程的组成

  1. PCB，进程控制块：

    1.1 进程描述信息：即PID（进程标识符），UID（用户标识符）

    1.2 进程控制和管理信息：进程的状态、优先级、地址、时间信息、信号量等

    1.3 资源分配清单：进程运行需要的各种指针、文件描述符等

    1.4 处理机相关信息：处理机上下文，包括各种寄存器的值（外部中断时操作系统保存的通用寄存器就是在这里）

  2. 正文段：二进制代码与全局变量、常量

  3. 数据堆段：动态分配的变量或数据

  4. 数据栈段：局部变量、传参（因为会递归调用，所以用栈存储）

线程

用户级线程与内核级线程

用户级：用户级线程切换开销小；允许每个进程自己定制线程的调度算法，管理灵活；不涉及内核，可在任何操作系统内运行

内核级：内核级线程有一个阻塞后可以调用其他线程继续执行；多核处理器允许统一进程的多线程并行

调度算法种类：

  1. 先来先服务（FIFS, First In First Serve）:

    按照到达时间的次序依次加入运行队列，该算法对于长进程较为友好，对于短进程不友好

  2. 最短作业优先调度算法（SJF, Shortest Job First)

    不考虑先后次序，只考虑任务量大小，仅从平均周转时间看来，该算法最为优秀，对短进程友好，长进程可能会饥饿

  3. 最短剩余时间优先调度算法（SRTF, Shortest Rest Time First）

    SJF算法的一种变形，抢占式，新算法到达后与正在运行的算法比较CPU执行的剩余时间，剩余时间长的排后面，短的先使用CPU

  4. 最高响应比算法（HRRN, Highest Response Ratio Next)

    Hi=Ti/ri=(ri+wi)/ri=1+wi/riH_i=T_i/r_i=(r_i+w_i)/r_i=1+w_i/r_iHi​=Ti​/ri​=(ri​+wi​)/ri​=1+wi​/ri​ 其中rir_iri​是需要在CPU上执行的时间，wiw_iwi​是任务自从到达之后未使用CPU的时间（等待时间）随着进程的等待时间增加，其响应比就越高，防止饥饿，且同等等待时间下CPU运行时间更少的响应比更高，让计算量较小的任务可优先进行计算

  5. 基于优先级的调度方法

    5.1 静态优先级：在进程或者作业被执行之前就确定了，不可调整

    5.2 动态优先级：计算综合考虑初始设定的优先级、任务的CPU执行时间、等待时间等，可动态调节

  6. 时间片流转调度算法（RR, Round Robin）

    将CPU的执行时间划分为若干个时间片，每个进程运行时都为其分配一个时间片，时间片用完后将会从CPU中剥夺出来，并将其排到就绪队列的尾部，等待下一次的调度

临界区管理方法

临界区使用原则

  1. 空闲让进：当一个临界区没有被其他进程占用，则在某进程执行想要访问临界区的代码时，临界区需要被其占用

  2. 忙则等待：若是该临界区已被占用，则该进程应当等待，直到空闲

  3. 有限等待：进程要在一个可以预见的时间等待，不能无止尽地等待临界区资源

  4. 《王道》：让权等待：进程不能进入临界区时要及时释放处理器，防止一直等待占用CPU资源

    《现代》：多种选一：有多个进程等待临界区时只能选择其中的一个进入，其余仍然为等待态

软件方法

Dekker算法

算法分析：

Dekker算法通过检测~~抢占~~的方式来实现临界区的访问，当且仅当该进程表达了访问意愿且另一个进程愿意让渡资格时才能够进行访问。但双层的while循环语句给计算机带来的计算开销略大

Peterson算法

算法分析：

相较于Dekker算法，Peterson算法只用了一次循环，应用检测~~谦让~~的方式实现临界区的访问，即当且仅当该进程声明访问意愿且对方进程无意愿或无资格时才能够进行访问，两进程同时运行时通常是最后一个让渡资格的进程被阻拦

硬件方法

  1. 关中断指令：在访问临界区时执行关中断指令，这样就不会被断进程导致临界区访问混乱，但不适用于多处理机（关中断只对进程本身的处理机有效），且单处理机下还会降低代码的并发性（无法中断切换进程了）

  2. TS(Test and Set)指令：也称(TSL, Test and Set Lock)，通过给临界区“上锁”的方式来进行互斥访问

  3. 数据交换(Swap / XCHG exchange)：逻辑上原理和TS指令相差不大，也是要循环检测是否上锁且每次检测都要将lock设置为true

    Swap 和 TSL 指令一样，都不满足 “让权等待” 的原则，占时无法进入CPU的进程会一直占用CPU，出于 “忙等”

信号量

信号量的数据结构
P、V操作

P操作：获取信号量，使信号量s.value -1 的操作，执行之后判断s.value大小，若小于0，则将该进程加入到等待队列中，否则，正常执行下一步骤

V操作：释放信号量，使信号量s.value+1 的操作，执行之后判断s.value大小，若大于0，则将从等待队列中唤醒一个进程，否则，正常执行下一步骤

经典进程互斥问题

哲学家就餐

问题描述：

圆桌上均匀坐着5名哲学家，每两名哲学家中间摆放一只筷子，共五根。每名哲学家面前有一碗米饭。哲学家只有思考与进餐两种状态：在思考时不会影响他人，在进餐时会拿起左右的筷子，只有拿到两根筷子后才能开始进餐，否则只能一直等待。进餐完毕后，哲学家会将筷子放回原处并继续思考

解决思路：

死锁的条件与预防

死锁的产生有四个条件，即互斥使用、不可抢占、请求保持、循环等待，但是互斥使用这一条件往往会使得临界资源管理更加困难，故从另外三个条件入手，破坏死锁产生的条件，从而预防死锁

破坏“不可抢占”

不可抢占，又称“不可剥夺”，即资源申请方无法从资源占有方处强行获取所需资源（都是进程），需要占有方资源释放。其破坏方法就是让进程申请资源后，若是申请的新资源无法立刻得到满足，则在将其转换为等待态前，将该进程已经占用的资源剥夺，交还给计算机

破坏“请求保持”

请求保持，指进程在已经申请使用过一次资源的前提下，又重新申请了新的资源，而对原本使用过、之后不会再使用的资源并不会主动释放，导致资源无效占用。其破坏方法为要求进程一次性将所有需要的资源申请并分配过去，进程执行完之后再一次性释放出来

这样的方法会导致资源闲置率高，利用率下降，且能够同时满足进程资源的数量大幅度减少

破坏“循环等待”

循环等待，即进程在等待之前的进程运行完毕后释放资源，之前的进程也在等待更前的进程释放资源，而最前的进程却又在等待新进程所占用的资源，形成“衔尾蛇”，哲学家进餐问题的死锁就是这样的情况。破坏该条件的方法就是为每个资源编号，并且要求进程对资源的申请只能由小到大，若是小序号资源已被占用，则先将其转化为阻塞态进行等待，加入资源1的等待序列

银行家算法 避免死锁

避免死锁，其核心逻辑就是分配资源前，先检测分配后是否存在可以让所有资源都能收回的安全序列，若存在则进行分配，否则拒绝分配。银行家算法的前提如下：

  1. 假设某进程能够顺利获得所有资源，其完成后就一定能够将所有资源返还

  2. 假设进程获得资源后，若不能满足申请的总资源数量，则无法完成，且完成之前不能归还已申请的资源

系统要在每次分配资源前都测试分配后的剩余资源能否满足剩余客户的需求，只要能够找到满足剩余资源需求的安全序列，就一定不会产生死锁，系统就处于安全状态

只要系统处于安全状态，就一定不会产生死锁，反之若是没有处于安全状态，也不一定会造成死锁，因为可能前提2中会有“惊喜”，即使没有满足总的资源数量，进程还是能够完成，并返还所占有的全部资源，即进程的实际资源需求小于其声明的资源需求

死锁的检测与接触

死锁的检测——进程-资源分配图

如图所示，该图展现了计算机中的资源分配情况，其中圆形表示进程，方形表示资源种类，P1→R1P_1\rightarrow R_1P1​→R1​ 表示P1向R1申请资源，R3→P1R_3\rightarrow P_1R3​→P1​ 表示R3已经分配给P1一份资源通过该图，我们就可以进行资源监测：

  1. 找到只有分配边，没有申请边的进程，将其边抹去，代表不再需要申请进程且进程运行完毕，释放占用的资源

  2. 将有申请边和空闲资源的资源进行分配，并将申请边更改为分配边

  3. 重复以上步骤，直至所有边都被抹除。若是执行上面两步仍然有边的存在，则说明出现死锁，否则就是暂时没有死锁，且进程的处理顺序就是安全序列

死锁的解除

资源剥夺：发现存在死锁后，将死锁环中的部分进程剥夺一定数量的资源（将分配边变为申请边），再分配给另外的资源，优先满足其中一个能够处理完毕，从而解除死锁

进程撤销：终止全部的死锁进程，让系统恢复正常状态，重新进行资源分配——改进：按照某种顺序逐个终止死锁进程，直到其他进程能够正常运行，产生安全序列

第三章 存储管理

知识点总结

分区

分区，学名连续分配，即为一个用户分配一个连续、完整的用户空间。连续分配方式包括单一连续分配、固定分区分配、动态分区分配

单一连续分配

在此方式下，内存分为用户区、系统区，系统区只能系统使用、用户区仅有一道用户程序，整个用户区内存空间被该程序独占

这种方式的优点是简单、无外部碎片、无序内存保护，可以使用覆盖技术，但只能用于单用户、单任务的操作系统，存储器利用率极低

固定分区分配

将用户内存空间划分为若干大小固定的区域，每个分区只装入一道作业，有空闲分区时就到外存的就绪作业队列选择大小合适的作业调入。该方法有两个不同的划分分区方式

  • 分区大小相等：所有分区都一样，程序小了浪费，大了装不下，缺乏灵活性

  • 分区大小不等：多个小分区，适量中分区，少量大分区

这样的分区方式存在两个问题：

  1. 可能有程序太大而无法放下，这时需要使用覆盖技术来使用内存空间

  2. 程序很少完全符合分区大小，往往装入的程序大小都小于该分区，容易产生内部碎片

动态分区分配

又称“可变分区分配”，在装入程序时根据进程实际需要动态地分配内存，使得分区大小正好符合进程需要

在装入或换入进程时，若存在多个足够大的空闲块，系统需要判断使用哪一个进行分配，这就涉及到以下几种动态分区分配算法：

  1. 首次适应(First Fit)：将空闲区以首地址递增的次序进行排序（用链表就是连接，表格就是排序，下同），每次分配内存，都从头开始顺序查找，将第一个满足内存需求（空闲区大小>=程序大小）的空闲区进行分配，并更新剩余的空闲区大小

  2. 临近适应(Next Fit)：又称“循环首次适应算法”，与其不同之处在于分配内存的起始点不是开头，而是上次查找结束的位置

  3. 最佳适应(Best Fit)：空闲区按照剩余空间大小递增进行排序，找到第一个满足要求且最小的空闲区进行分配，尽可能留大的，容易产生外部碎片

  4. 最坏适应(Worst Fit)：空闲区按照剩余空间大小递减进行排序，找到第一个满足要求且最大的空闲区进行分配，外部碎片相对较少，但很快就没有足够大的内存块，导致大进程难以进入

分页

逻辑页面

逻辑页面要点主要在把握逻辑地址的拆分与转化，在已知页面大小为4KB4KB4KB（212B2^{12}B212B）的情况下，其逻辑地址的后12位就用于表示页内地址，再看逻辑地址的总位数（通常就是内存的位数），若总共是32位，则前20位则表示逻辑页面的页号

物理页框

页框又称内存块、物理页面、物理块，是在内存中实际存储数据资源的物理分区，内存块大小==页面大小

页表

将逻辑页面号和物理页框号对照起来的表单，页表项只需要保存物理页框号，逻辑页面号就是其在表格中的id(0起)。因此，知道了内存块的大小，我们就能知道一个内存中最多可以分为多少个页框——即内存块号的最大总数=2k/2m=2n=2^k/2^m=2^n=2k/2m=2n

其中k表示内存的支持位数，生活中常见的是64位，题目中常见的是32位；m表示页内地址的位数，即上面逻辑页面部分的页面大小转换后的次幂；n表示这个数量下的内存块号，其实就相当于是逻辑页面的页号大小。

由于一个数字只占用1bit=1/8B，故一个页表项占用空间为⌈n8⌉B\left \lceil \frac n8 \right \rceil B⌈8n​⌉B（不满1B的向上取整），页表的最大大小就是$2^n\times\left \lceil \frac n8 \right \rceil $B

二级页表

上面计算可知，直接使用页表的话长度会很大，因此可以使用分层的方式进行页表查询，即为页表再创建一个页表，用于更快找到页表的位置。我们将最外面的称为一级页表，较内的称为二级页表，此时的地址查找顺序就成了一级页表—二级页表—物理页框，逻辑地址的结构也就成了【一级页号，二级页号，页内偏移】，首先使用一级页号从一级页表找到对应的二级页表地址，再用二级页号在二级页表中找到对应的物理页框地址，最后使用页内偏移找到准确的物理地址

逻辑地址的转换

  1. 根据逻辑地址计算出页号、页内偏移量

  2. 根据题目信息判断页号是否越界

  3. 查询页表，找到页号对应的内存块号*——第一次进入内存*

  4. 用内存块号和页内偏移量找到具体物理地址

  5. 访问内存单元*——第二次进入内存*

    对于二级页表来说，转换步骤就在2和3之间增加了一步，2.5 查询一级页表，找到对应的二级页表地址；内存访问次数相对应地增加一次

快表

根据局部性原理，为了更好地提高系统的运行效率，改善用户的使用体验，可以在CPU的MMU内的相联存储器中存放一个页表，在CPU中进行页表查询可以极快地获得信息，大幅缩短访问时间，提高系统效率

  • MMU，Memory Management Unit，这是系统的内存单元，负责处理CPU的内存访问器请求

  • 相联存储器，MMU中的高速缓冲存储器，用于存放最近访问过的部分页表，即快表

局部性原理

一段时间内，程序的执行所存在的高度关联的时空局部性

  • 时间局部性：程序中往往会存在大量的循环语句或者高频重复调用，某个指令被访问过后，近期内再一次被执行或者访问的概率较高

  • 空间局部性：当一个存储单元被访问时，近期内的其自身其附近的存储单元被访问的概率也会更高，可能这段时间都会访问这一块

增加快表后访问流程

  1. 根据逻辑地址提取出逻辑页号与页内偏移量

  2. 将逻辑页号与快表中保存的页号进行比对，若有，则直接读取页框号，否则进行下一步

  3. 进入内存，访问内存中的页表，同时将访问到的页表项添加进入快表。若快表已满，则根据页面置换算法置换出其他页表项

  4. 根据读取到的页框号和页内偏移量，找到对应的物理地址，访问内存读取数据

分段

与分页相似，分段就是将程序按照各自的体量分配大小不等的程序段，这样有效避免了分页中产生的内部碎片问题，分段存储的管理机制与分页也十分相似，逻辑地址也是分为段号和段内地址

内部碎片

即分配内存过程中页面划分的空间有剩余，用不到的地方，但是每一块空间是一个整体，不能分配给其他需要的程序使用，导致每个程序给分的空间最后都会出现小块的空白但无法利用的空间——分配空间内部的空间浪费

外部碎片

即分配的空间之间存在着一个个小的空间，这些空间大小无法被其他进程使用，又无法分配出去，长期的积累也是很大的一部分空间浪费。外部碎片可以使用**紧凑（拼凑,Compaction）**的技术解决——分配空间之间的空间浪费

分页和分段的比较

  1. 对程序的划分和内存分配方式不同：分页按照固定大小划分程序和内存，分段按照程序中各逻辑段的大小划分程序和内存

    但二者在分配时，页内/段内的地址都是连续的，页间/段间的物理地址可能是分散的

  2. 在逻辑地址映射物理地址时，页的逻辑地址需要经过页号转化成页框号，页框号对应到物理地址的过程，而段的起始地址就是物理地址，直接配合段内偏移使用即可

  3. 分页便于管理内存，解决了内存的外部碎片问题，但不容易实现内容共享，特别是代码共享，以及动态链接——因为大小都固定了

    分段可以动态申请内存，统一管理和使用，可以内容共享、动态链接，但容易出现外部碎片，不易动态扩展

页面置换算法

缺页中断

在使用快表，或者虚拟内存的时候，会出现准备的页面不足，导致需要重新调入的情况，这就是缺页中断，此时会重新寻找页面并使用页面置换算法进行置换，缺页中断率的算法就是缺页中断率=访问失败重新调入的次数/（成功访问的次数+访问失败重新调入的次数），缺页率越低，说明使用的页面置换算法越好

页面置换算法

  1. 理想页面置换算法（OPT）

    这是最理想也是最不可能实现的置换算法，当需要发生置换时，观察后续的访问队列，找到以后不在访问或访问时间间隔最长的页面

  2. 先进先出页面置换（FIFO）

    记录下每个页面被调入的时间，到需要置换时选择被调入最早的进行置换*——是唯一一个有Belady异常算法（增大容量，缺页次数不减反增）*

  3. 最近最久使用置换（LRU）

    在页面每次被使用时进行更新，当需要被置换时查找距离上次使用时间最长的页面并进行置换

  4. 时钟页面置换算法（CLOCK）

    为内存中的页面设置一个循环队列，并为表中每项页面设置访问位，若被访问过则设置为1。若需发生置换则根据指针逐个进行检查：检测到1，则将其访问为置0，跳转下一项；检测到0，则对其进行置换，置换后也访问为置1，指针调到下一项

    选择一个页面进行置换最多需要遍历两轮

  5. 改进时钟页面置换（改进CLOCK）

    在原本的时钟页面置换算法的基础上，再添加一个修改位，若被修改则置1，仅访问而未被修改则置0；未被修改的页面被置换后不需要返回内存，直接覆盖即可。访问位和修改位的情况在下面用（访问，修改）样式表示。改进后的工作流程如下

    第一轮：从当前位置开始扫描到第一个（0,0）的页面用于替换——未被访问未被修改

    第二轮：若上一轮未找到可替换项则开始，查找第一个（0,1）的页面用于替换，且若扫描到（1，x）的页面则将访问为置0——未被访问被修改——唯一一个需要修改的轮次

    第三轮：若上一轮未找到可替换项则开始，查找第一个（0,0）的页面（即第一轮的时候的1,0）用于替换——被访问未被修改

    第四轮：若上一轮未找到可替换项则开始，查找第一个（0,1）的页面（即第一轮的时候的1,1）用于替换——被访问被修改

    选择一个页面置换最多需要进行四轮扫描

简答题

  1. 在一个分也许你存储管理系统中国，用户编程空间为32个页，页长为1KB，内存为16KB。若用户程序页长为10页，已知逻辑页号0,1,2,3已分配到页框8,7,4,10，请将虚拟地址 0AC5H 和 1AC5H 转换为对应的物理地址

    分析：页长1KB，则虚拟地址后10位都是页内偏移，往前都是逻辑页号。虚拟地址转换为物理地址，页内偏移不需要变，把逻辑页号部分换成物理页号即可

    用户变成空间为32个页，为252^525,右起第11~第15为逻辑页号，第16位填充0

    0AC5H转换成二进制为 0000 10,10 1100 0110 其中后十位是页内偏移，逻辑页号为2，分配页框号为4，对应物理地址为0001 0010 1100 0110，十六进制为12C5H

    1AC5H转换为二进制为 0001 10,10 1100 0110 其中后十位是页内偏移，逻辑页号为8，未分配页框

  2. 描述存储保护和地址越界中断机制

    存储保护，是在支持多进程并发的存储管理系统中，保障每个进程所占用内存空间的代码、数据等信息的安全性和可靠性，主要由地址越界保护和操作越权保护

    地址越界中断就是越界保护的一种，系统会为每个进程分配逻辑桑相对独立、封闭的空间，通过设置上限寄存器和下限寄存器来检测是否发生越界，若发生越界则中断该进程返回相应的错误信息

第四章 IO设备管理

知识点总结

IO设备的控制

按照IO控制器功能的强弱及其与CPU协同方式的不同，可将控制方式分为4中：询问、中断、DMA、通道，方法的不断改进，是为了提高CPU和外围设备之间的并行程度，让CPU能花费更少的时间在IO控制中，多做计算

基于询问的设备控制

又称程序直接控制方式，CPU全程参与整个传输过程，工作流程如下：

  1. 系统利用设置的计数器指明一共需要传输的数据量

  2. 设置内存的缓冲首地址，指明数据在内存中存放的起始位置

  3. 发出启动IO设备的命令，反复检查IO设备是否准备就绪，即“询问”

  4. 若IO设备就绪，传送一个字（2B）或字节（1B=8bit）

  5. 修改内存地址，指针指向下一个地址准备写入

  6. 修改计数器，直至为0表示全部传输完毕，否则回到步骤3继续执行

该算法有一下几个特点（缺陷）：

  1. 传输单位是字或字节，需要反复启动IO设备，知道传送结束，效率低下

  2. 一旦启动IO设备，CPU需要反复确认IO设备的就绪状态，这样会暂停原程序的执行，占用CPU资源

  3. IO设备就绪后，CPU全程负责数据传输，即读入数据写到内存，或将内存数据传输到IO

基于中断的设备控制

该方法解决了反复询问IO设备是否就绪而浪费的资源，改进后的原理如下：

  1. CPU启动外设后，不再反复询问，而是继续执行计算

  2. IO设备就绪后，会向CPU发出中断通知

  3. IO设备就绪后的数据传输，CPU仍要全程负责

基于DMA的设备控制

基于终端的设备控制方法仍需要CPU全程负责数据传输，效率还是差点。该方法在系统中增加了一个DMA(Direct Memory Access，直接存储器访问)控制器，让其作为CPU的助手完成数据的输入输出。工作原理如下：

  1. CPU启动IO设备后，向其发送传送一个数据块的命令

  2. 让DMA完成数据块的实际传输

  3. 传输完成数据块后，CPU再来完成结束传输的相关工作

该方法使用DMA控制器替CPU完成了数据的存入和取出，核对数据计数器的任务也由DMA完成，减少了CPU的传输工作，令其只需要负责每个数据块传输前和结束时的工作，同时实现了传输单位到数据块的转变

基于通道的设备控制

DMA中CPU还需要负责每个数据块传输前和结束时的工作，若是数据块多，也是一种时间成本。通过建立通道，可以让CPU只用在整个数据传输过程的开始前和结束时参与线管工作即可，其余时间可完全投入到计算中

通道又称为输入输出处理器（I/O Processor），相当于一个只具备数据传输功能的弱化版CPU，可以成批量地传输数据块——即一次传输一组数据。

缓冲

由于IO设备的速度与计算机的计算速度严重不匹配（CPU很快，打印机很慢），且每一次使用IO设备都需要一次中断，对系统的性能影响较大，因此系统设计人员在内存中开辟了一个特定的缓冲区域，作为CPU与IO设备的数据交换中介，

总体工作流程

  • IO设备将数据输入到缓冲区，进程执行到需要IO设备的数据后，系统将对应的数据传送给进程

  • 进程将数据输入到缓冲区，直到装满或者输入完成，随后待指定的IO设备空闲后再将缓冲区数据传输到IO设备

多种缓冲技术

分析多种缓冲技术时，我们结合其工作时的情景进行分析，首先假设数据从磁盘到缓冲区的传输时间为T，从缓冲区到CPU的世家为M，CPU计算处理数据的时间为C

  • 无缓冲技术（作为空白对照组）

    没有使用缓冲技术时，数据直接从硬盘传输到CPU，CPU处理完数据才能进行下次传输，故总耗时为T+M+CT+M+CT+M+C，其中 T+M 表示数据从磁盘经过内存，最后到CPU的总时间

  • 单缓冲技术

    即仅有一个缓冲区进行数据传输，当缓冲区将数据传送给CPU后，在CPU进行数据分析时，又可以继续进行下一次数据的传输，此时传输过程的总耗时为Max(C,T)+MMax(C,T)+MMax(C,T)+M

  • 双缓冲技术

    在使用单缓冲的时候可以发现，CPU在数据传输到缓冲区M这一过程中处于空闲态，为了提高CPU的利用率，引用了双缓冲技术，两个缓冲区轮流接收数据并向CPU进行传输，极大提高利用率，此时处理一块数据用户为Max(C+M,T)Max(C+M,T)Max(C+M,T)

  • 循环缓冲技术

    循环缓冲包含了许多大小相等的缓冲区，且每个缓冲区都有指向下一个缓冲区的指针，最后一个缓冲区指向第一个，形成一个循环。此外还有两个负责输入与输出的指针 in 和 out ，in指向还可以继续输入数据的未满缓冲区，out指向可以向外输出数据的已满缓冲区——这里的入指的是向CPU输入，让CPU读取数据

  • 缓冲池技术

    缓冲池技术则是在上面的基础上进一步拓展，由多个系统公用的缓冲区组成，按照使用状况分为空缓冲队列、待输入队列、待输出队列，同时其中的缓冲区可以分为：收容输入(h-in)、提取输入(s-in)、收容输出(h-out)、提取输出(s-out)。实际情况来说，输入进程需要输入数据时，就从空缓冲队列队首摘一个空缓冲区，作为收容输入缓冲区，插入带输入队列队尾；计算进程需要输入的数据时，就从待输入队列的提取输出缓冲区摘一个缓冲区，用完后挂到空缓冲队列中。输出同理

硬盘

硬盘的结构

硬盘是最为常见的设备结构，机械硬盘是由多个盘面叠加而成的，多个盘面依靠中间轴心一起转动，每个盘面上方都有一个磁头，形成一一对应的关系。所有刺头都固定在一个驱动臂上，在同一时间进行移动。

盘面上读、写磁头的轨道称为磁道，相同半径的所有磁道形成的柱状边称为柱面，一条磁道可以根据中心轴的射线划分为多个物理块，即扇区，为了提高文件的读写效率，通常文件数据信息都是存储在同一柱面的不同磁道上，以此实现并行读取/写入操作，减少驱动臂的移动距离和次数。可以通过磁道号/柱面号、磁头号/盘片号、块号/扇区号，这三组数据来唯一确定一个硬盘位置信息

硬盘的数据访问时间

访问一个硬盘的数据，需要经过：

磁头驱动臂移动磁头到指定磁道/柱面——中心轴旋转，让磁头指到指定扇区——将数据读出或写入磁盘

这三个步骤，每一个步骤的时间都需要单独计算，分别是：寻道时间、旋转延迟时间、数据传输时间

  • 寻道时间

    是启动磁头驱动臂的时间s和磁头移动n条磁道所用时间之和，T=m×n+sT=m\times n+sT=m×n+s，其中m表示磁盘移动到相邻磁道所用时间

  • 旋转延迟时间

    旋转找到扇区的时间，通常用 旋转一周时间/2 的表示平均旋转延迟时间

  • 数据时间

    这一时间通常无法计算，由题目给出，往往也是耗时最短的

硬盘驱动臂调度算法

  1. 先来先服务

    按照各个进程提交的请求时间先后顺序来决定提供服务的次序不会发生磁臂粘着

  2. 最短查找时间

    先执行查找之间最短的磁盘请求——即距离当前磁盘最近的磁道，但是会导致饥饿问题，即有较远的请求可能长时间无法被服务

    先来先服务算法和最短查找时间算法都是与移动方向无关的算法，只在乎请求本身的属性（到达时间、请求位置等）

  3. 电梯调度

    磁头选择距离移动方向最近的柱面（通常是往大的跑），直到该方向上没有访问请求时，才会转换方向往前往另一边的最近柱面

  4. 扫描

    与电梯调度类似，磁头往前进方向一直到最后一个柱面，然后掉头反向移动，在最大磁道号和最小磁道号之间来回移动

    电梯调度只走到最远的请求，扫描是走到最远的柱面

  5. 单向扫描

    从扫描算法中优化，总是从0号柱面向最大号柱面进行扫描，到头后直接返回0号柱面重新开始

  6. 分部扫描

    为了解决上述算法中除了FIFS以外都有磁臂粘着现象这一问题，分布扫描算法采用了以下处理策略：

    1. 将请求队列分成长度若干的子队列

    2. 磁臂按照先来先服务算法一次处理这些子队列

    3. 处理一个队列时又按照扫描算法依次处理每一个磁道访问请求

    4. 出现新请求后，系统将新请求进程放入其他队列，避免发生“磁臂粘着”

    与时间片算法类似，分布扫描算法的子队列长度也不宜过长/过短，长了就是扫描，短了就是先到先服务

简答题

  1. 操作系统的设备管理模块包含哪些主要机制以提升性能？

    在操作系统的设备管理模块中，为了解决反复询问IO设备就绪的问题，采用了基于中断的设备控制，只有当IO设备准备就绪时才会发出中断，让CPU进行处理

    为了解决单个字节传输以及CPU控制数据传输问题，使用了DMA控制器进行数据块的传输，数据块传输开始前和完成一个数据块的传输时才需要CPU进行IO操作的处理

    为了解决CPU在每个数据块传输前后都要CPU参与的问题，采用了通道技术，让CPU只需要在整个数据的传输前和结束时参与相关工作即可

  2. 输入数据时，若是采用中断控制方式，系统工作包含哪些步骤？

    使用基于中断的设备控制方式，系统的主要工作流程如下：

    1. 系统利用计数器获取总共传输的数据量

    2. 设置内存的缓冲首地址

    3. CPU开始处理其他计算任务，直到IO设备发出中断

    4. CPU进行数据传送

    5. 修改内存指针，指向下一个地址

    6. 修改计数器-1，直到计数器为0,否则返回步骤3继续执行
